Data flow: The VAE Encoder (training only) produces the clean latent z₀ which gets noised to zt — the input to the U-Net. CLIP text embeddings enter the U-Net via cross-attention (a separate input, not connected to the VAE). At inference, zt starts as pure noise and the VAE Encoder is not used.

The Three Components

1 CLIP Text Encoder

  • Frozen OpenAI CLIP ViT-L/14 text encoder
  • Tokenizes prompt → max 77 tokens
  • Outputs sequence of 768-dim embeddings
  • Fed to U-Net via cross-attention at multiple resolutions
  • Not trained during SD training — used as-is
tokens = tokenizer(prompt) # [1, 77] text_emb = text_encoder(tokens) # [1, 77, 768]

2 U-Net (Conditional)

  • Same encoder-bottleneck-decoder structure as basic U-Net
  • Operates on 64×64×4 latents (not pixels)
  • ~860M parameters
  • Cross-attention layers inject text conditioning:
    Q = latent features, K/V = text embeddings
  • Time embedding via sinusoidal + MLP (same as before)
noise_pred = unet(z_t, t, text_embeddings) # z_t: [1, 4, 64, 64], t: scalar, text: [1, 77, 768]

3 VAE (Encoder + Decoder)

  • Compresses 512×512×3 → 64×64×4 (8× spatial reduction)
  • Trained separately, frozen during diffusion training
  • Encoder: used only during training (encode real images to latents)
  • Decoder: used at inference (latents → pixels)
  • Scaling factor: latents are divided/multiplied by 0.18215
# Training: latent = vae.encode(image).latent_dist.sample() * 0.18215 # Inference: image = vae.decode(z_0 / 0.18215)

+ Classifier-Free Guidance (CFG)

  • At each step, run U-Net twice:
    once with text (conditional), once without (unconditional)
  • Interpolate to amplify the text influence:
ε = ε_uncond + guidance_scale × (ε_cond − ε_uncond)
  • scale = 1.0: no guidance (ignores text)
  • scale = 7.5: standard (good balance)
  • scale > 15: oversaturated, artifacts

Cross-Attention: How Text Guides the U-Net

The only difference from an unconditional U-Net is the addition of cross-attention layers. In self-attention, Q/K/V all come from the image features. In cross-attention, Q comes from the latent features but K and V come from the text embeddings.
Latent features (H·W × C) Text embeddings (77 × 768) │ │ ▼ ▼ Q = W_q · latent K = W_k · text V = W_v · text │ │ └──────────────┬─────────────────────┘ ▼ Attention = softmax(Q · Kᵀ / √d) · V │ ▼ Text-guided features

This lets each spatial position in the latent "look at" the text tokens and decide which words are relevant for that location. E.g., the top-left region might attend strongly to "sky" while the bottom attends to "ground".
